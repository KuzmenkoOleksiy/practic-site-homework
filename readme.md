# Технічне завдання: Адаптив хедера та секції Hero

## Мета

Забезпечити коректне та гармонійне відображення **Header** і секції **Hero** на всіх вказаних розмірах екрана без появи горизонтального скролу.

---

## Точки перелому

- **320px** — базовий мобільний вигляд
- **480px**
- **768px**
- **1200px**

---

## Загальні вимоги

1. Розробити повноцінний адаптив хедера та секції Hero.
2. На кожному брейкпоінті елементи мають коректно перебудовуватися та зберігати логічну ієрархію.
3. На сторінці не повинно бути горизонтального скролу:
   - Контент не виходить за межі ширини екрану.
   - Відсутні фіксовані ширини, що можуть викликати переповнення.
4. Використовувати сучасний підхід до адаптиву:
   - Flexbox
   - `max-width`, `object-fit`, коректні відступи.
5. Адаптив реалізувати через медіа-запити з урахуванням усіх точок перелому.

---

## Вимоги до Хедера

- Логотип адаптивний, не виходить за межі контейнера.
- Навігація перебудовується залежно від ширини:
- Кнопки та елементи керування не зливаються, зберігають відступи.
- Висота хедера стабільна і не перекриває вміст.

---

## Чекліст перед здачею

- [ ] Жодного горизонтального скролу.
- [ ] Хедер відображається коректно на всіх брейкпоінтах.
- [ ] Бургер працює та не перекриває контент.
- [ ] Hero виглядає гармонійно і читається.
- [ ] Зображення не виходять за межі.
- [ ] Всі відступи та типографіка коректні на різних ширинах.
